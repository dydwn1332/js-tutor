    • 코드 설명

    먼저 집중 5개 섹션 과정에서 JS의 모범 사례를 배우십시오. 주제에는 데이터 캡슐화, 클로저, 바인딩, 상속 및 이름 간격이 포함됩니다. 다른 브라우저에서 JS 문제를 디버그하고 성능을 개선하는 방법이 잘 알려져 있지 않지만 유용한 언어 기능을 알아보세요. 타사 JS 라이브러리를 사용하여 대화형 웹 페이지를 만듭니다.

    실습실에는 컴퓨터가 제공되지만 수업 중 연습을 위해 노트북을 가져오는 것이 좋습니다.

    사전 학습

    • JS의 변수, 데이터 유형, 제어 흐름 및 기본 함수 사용법에 대한 이해
    • HTML에 대한 강력한 중급 지식 및 최소한 CSS의 기초
    • 기본 jQuery 지식(요소/클래스 추가/제거와 같은 DOM 상호 작용)

    이러한 사전 지식이 없으면 꽤 길을 잃을 것입니다. 복습이 필요하면 초급 자료를 살펴보십시오.

    코스 개요

    JS의 뉘앙스, 프로토타입 상속이 고전 상속과 어떻게 비교되는지, 동적 및 복잡한 웹 애플리케이션을 구축하는데 이것이 어떻게 사용될 수 있는지 자세히 알아볼 것입니다. jQuery 및 BackboneJS와 같은 최신 도구에 대해 논의하지만 학생들은 이러한 프레임워크의 구성 요소를 배우고 이 과정을 마친 후에는 내부에서 일어나는 일을 이해할 수 있습니다. 대부분이 Node.js, Phonegap 등과 같은 다른 시스템에 적용되지만 초점은 브라우저 개발에 있습니다. 다루는 주제는 다음과 같습니다.

    • 캡슐화, 클로저 및 범위
    • 전형적인 상속과 프로토타입 상속
    • 이벤트 루프
    • AJAX와 JSONP
      • 로컬
      • 원격(예를 들어, Foursquare)
    • 스크레치로부터 MVC-스타일 모델 생성(a'la Backbone.js)
    • 테스트 및 의사 코드 지향 개발

    주제들은 deck에서 사용할 수 있는 라이브 코드 예제/슬라이드를 통해 시연됩니다. 추가 연습은 수업에서 완료됩니다.

    자세한 배경은 여기를 참조합니다.

    워크플로

    GitHub 데스크톱을 사용하는 경우에는 다음 일반 지침이 도움이 됩니다.

    • https://guides.github.com/activities/forking/
    • https://help.github.com/desktop/guides/contributing/

    Edit->Automatically Sync after Committing가 권장됩니다. 단계는 다음과 같습니다.

    1. exercise/project의 저장소 분기(github.com/advanced-js에 있음.).
    2. 저장소의 컴퓨터 복사
    3. 브라우저에서 index.html을 열고, 개발자 도구 열기
    4. 파일 수정을 통한 솔루션 완료
    5. index.html 페이지를 새로 고침하여 결과를 확인하고 반복함.
    6. 코든 코드가 멋지게 커밋되었는지 확인
    7. GitHub에 밀어넣기/동기화
    8. 원래 리포지토리에서 끌어오기 요청 생성; 모든 과제는 달리 명시되지 않는 한 다음 수업이 시작될 때 마감됩니다.
    9. 종료 날짜까지 수정 사항 및 개선 사항을 계속 밀어 넣을 수 있습니다. 끌어오기 요청에 댓글을 추가하여 업데이트 되었음을 알려주십시오.

    끌어오기 요청이 생성되면 "Travis CI 빌드가 진행 중입니다."라는 메시지가 표시되어야 합니다. 이는 솔루션이 구문 오류에 대해 자동으로 검사되고 있음을 의미합니다. 이 빌드가 실패하면(빨간색 X가 표시됨) 세부 정보 링크를 클릭하고 맨 아래로 스크롤하여 오류가 무엇인지 확인하십시오. 해당 요구사항 아래의 문제를 해결하고 변경 사항을 밀어주십시오.

    피드백은 끌어오기 요청에서 진행할 예정이니 여러분의 생각과 질문에 답해주세요! 막혀서 질문을 하고 싶은 경우 진행 중인 작업으로 일찍 끌어오기 요청을 여는 것을 환영합니다. 솔루션은 http://USERNAME.github.io/EXERCISE에서도 사용할 수 있습니다.

    요구 사항

    다음은 실생활에도 적용됩니다.

    • Travis CI 빌드는 다음을 포함하여 통과되어야 합니다.
      • 모든 HTML 파일은 W3C Markup Validation을 통과해야 합니다.
      • 모든 작성된 JS는 JSHint를 통과해야 합니다.

      • 함수들은 짧아야 함(참고: Sandi Metz의 개발자들을 위한 규칙).
      • 가독성을 위한 최적화
        • 프로그램은 사람들이 읽을 수 있도록 작성되어야 하고, 기계가 실행하기 위해서는 부수적으로만 작성되어야 합니다." - 해롤드 아벨슨
      • 특히 콜백이 두 줄 이상인 경우 다른 함수 내에서 익명 콜백을 사용하지 마십시오.
      • 프로젝트의 경우 객체 지향 프로그래밍을 사용합니다.
    • 모든 차용된 코드에는 적절한 주석을 달아야 합니다.

표절 문제

표절을 매우 심각하게 여기며 사기의 한 형태로 간주합니다. 여기서 표절이란 다른 사람의 작품에서 의역된 구절 또는 다른 누군가가 구두 또는 서면으로 수집, 구성 및 보고한 사실이나 아이디어 표절은 사실 문제이지 학생의 의도가 아니므로 다음 사항이 중요합니다. 출처에 대한 확인은 정확하고 완전하며, 의식적으로 속이려는 의도가 없더라도 적절한 확인을 하지 않는 것은 표절에 해당합니다.

권고 사항

아이디어나 코드를 재사용하고 구축하는 것은 현대 소프트웨어 개발의 주요 부분입니다. 전문 프로그래머로서 처음부터 아무것도 작성하지 않을 것입니다. 이 강좌는 모든 솔루션이 공개되도록 구성됩니다. 동료의 작업에서 배우도록 권장됩니다. 단순히 솔루션을 복사하여 붙여넣는 사람들을 사냥하지 않을 것입니다. 왜냐하면 그들은 스스로에게 도전하지 않고 이 수업을 듣는 데 시간과 돈을 낭비하고 있기 때문입니다.
