그리고 내 심장은 내 필름처럼 반투명한 피부 아래에서 새빨간 빛으로 빛나고 그것들은 내가 돌아오도록 10cc의 JavaScript를 관리해야 합니다. (나는 혈액의 독소에 잘 반응합니다.) 그 물건은 복숭아를 아가미 밖으로 걷어차 버릴 것입니다!

_why, Why's (Poignant) Guide to Ruby

이 페이지에서는 실제로 프로그래밍 이라고 부를 수 있는 일을 시작할 것입니다. 지금까지 본 명사와 문장 조각을 넘어 의미 있는 산문을 표현할 수 있는 수준까지 JavaScript 언어의 명령을 확장할 것입니다.

식과 문장

앞 페이지에서 새로운 값을 얻기 위해 값을 만들고 여기에 연산자를 적용했습니다. 이와 같은 값을 생성하는 것은 모든 JavaScript 프로그램의 주요 내용입니다. 그러나 그것이 유용하려면 더 큰 구조로 짜여져야 합니다. 이것이 여기에서 취급할 내용입니다.

값을 생성하는 코드 조각을 식이라 합니다. 문자 그대로 쓰여진 모든 값(22 또는 "psychoanalysis"와 같은)은 표현식입니다. 두 개의 표현식에 적용된 이항 연산자 또는 하나에 적용된 단항 연산자와 마찬가지로 괄호 사이의 표현식도 표현식입니다.

이것은 언어 기반 인터페이스의 아름다움의 일부를 보여줍니다. 표현식은 인간 언어의 하위 문장이 중첩되는 방식과 유사한 방식으로 다른 표현식을 포함할 수 있습니다. 하위 문장에는 자체 하위 문장이 포함될 수 있습니다. 이를 통해 임의로 복잡한 계산을 설명하는 표현식을 작성할 수 있습니다.

표현식이 문장 조각에 해당하는 경우 JavaScript문은 전체 문장에 해당합니다. 프로그램은 명령문의 리스트입니다.

가장 간단한 종류의 명령문은 뒤에 세미콜론이 있는 표현식입니다. 이것은 프로그램입니다.

1;
!false;

경우에 따라 JavaScript에서는 명령문의 끝에서 세미콜론을 생략할 수 있습니다. 다른 경우에는 거기에 있어야 합니다. 그렇지 않으면 다음 행이 동일한 명령문의 일부로 처리됩니다. 안전하게 생략할 수 있는 경우에 대한 규칙은 다소 복잡하고 오류가 발생하기 쉽습니다. 따라서 이 페이지에서 세미콜론이 필요한 모든 명령문은 항상 세미콜론을 사용합니다. 적어도 세미콜론 누락의 미묘함에 대해 더 많이 배울 때까지는 동일한 작업을 수행하는 것이 좋습니다.

결합(Bindings)

프로그램은 내부 상태를 어떻게 유지하는가? 사물을 어떻게 기억하는가? 이전 값에서 새 값을 생성하는 방법을 살펴보았지만 이전 값은 변경되지 않으며 새 값은 즉시 사용되어야 하며 그렇지 않으면 다시 소멸됩니다. 값을 잡고 유지하기 위해 JavaScript는 바인딩 또는 변수라는 것을 제공합니다.

let caught = 5 * 5;

특수한 단어 (keyword) let는 문장이 바인딩을 정의할 것이라는 것을 나타냅니다. 그 뒤에는 바인딩 이름이 오고, 즉시 값을 지정하려면 = 연산자와 표현식이 옵니다.

이전 문장은 caught라는 바인딩을 생성해서 이를 사용하여 5에 5를 곱해 생성된 숫자를 파악합니다.

바인딩이 정의된 후 해당 이름을 표현식으로 사용할 수 있습니다. 이러한 표현식의 값은 현재 바인딩이 보유하고 있는 값입니다. 다음은 예입니다.

let ten = 10;
console.log(ten * ten);
// → 100

바인딩이 값을 가리킬 때 해당 값에 영원히 묶여 있다는 의미는 아닙니다. = 연산자는 자신의 현재 값에서 분리하고 새로운 지점이 기존의 바인딩을 언제든지 사용할 수 있습니다.

let mood = "light";
console.log(mood);
// → light
mood = "dark";
console.log(mood);
// → dark

바인딩을 상자가 아니라 촉수로 상상해야 합니다. 값을 포함하는 것이 아니라 값을 참조할 수 있습니다. 프로그램은 여전히 참조되는 값에만 접근할 수 있습니다. 무언가를 기억해야 할 때 그것을 붙들기 위해 촉수를 키우거나 기존 촉수 중 하나를 그것에 다시 부착합니다.

다른 예를 살펴보겠습니다. Luigi가 여전히 빚지고 있는 금액을 기억하기 위해 바인딩을 만듭니다. 그리고 그가 35달러를 갚았을 때, 이 바인딩에 새로운 값을 부여합니다.

let luigisDebt = 140;
luigisDebt = luigisDebt - 35;
console.log(luigisDebt);
// → 105

값을 주지 않고 바인딩을 정의하면 촉수는 잡을 것이 없으므로 허공에 끝납니다. 빈 바인딩의 값을 요청하면 undefined 값을 얻게 됩니다.

단일 let 문장은 여러 바인딩을 정의할 수 있습니다. 정의는 쉼표로 구분해야 합니다.

let one = 1, two = 2;
console.log(one + two);
// → 3

단어 var과 const 또한 바인딩 생성에 사용될 수 있습니다(let와 유사한 방식으로).

var name = "Ayda";
const greeting = "Hello ";
console.log(greeting + name);
// → Hello Ayda

첫 번째 var ("variable"에 관한 약어)은 2015 이전의 자바스크립트에서 선언되는 바인딩 방식입니다. 다음 장에서 소개되는 let와 차이가 있는데 일단 같은 작업을 한다고 할 수 있습니다. 앞으로는 거의 사용되지 않을 것입니다.

const는 상수 선언을 나타냅니다. 생존하는 동안 동일한 값을 가리키는 상수 바인딩을 정의합니다. 나중에 쉽게 참조할 수 있도록 값에 이름을 지정하는 바인딩에 유용합니다.

바인딩 이름

바인딩 이름은 모든 단어가 될 수 있습니다. 숫자는 바인딩 이름의 일부일 수 있습니다(예를 들어, catch22는 유효한 이름). 그러나 이름은 숫자로 시작해서는 안 됩니다. 바인딩 이름에는 달러 기호($) 또는 밑줄( _)이 포함될 수 있지만 다른 구두점이나 특수 문자는 포함되지 않습니다.

let와 같이 특수한 의미를 가진 단어들은 keywords이고, 바인딩 이름으로 사용할 수 없습니다. 또한 바인딩 이름으로 사용할 수 없는 향후 버전의 JavaScript에서 사용을 위해 예약된 단어가 많이 있습니다. 키워드와 예약어의 전체 리스트는 다소 많습니다.

break case catch class const continue debugger default
delete do else enum export extends false finally for
function if implements import interface in instanceof let
new package private protected public return static super
switch this throw true try typeof var void while with yield

환경

주어진 시점에 존재하는 바인딩 및 그것의 값 컬렉션을 environment라고 합니다. 프로그램이 시작될 때 이 환경은 비어 있지 않습니다. 여기에는 항상 언어 표준의 일부인 바인딩이 포함되며 대부분의 경우 주변 시스템과 상호 작용하는 방법을 제공하는 바인딩도 있습니다. 예를 들어, 브라우저에는 현재 로드된 웹사이트와 상호작용하고 마우스 및 키보드 입력을 읽는 기능이 있습니다.

함수

기본 환경에서 제공되는 많은 값에는 함수 타입이 있습니다. 함수는 값으로 래핑된 프로그램 조각입니다. 이러한 값은 래핑된 프로그램을 실행하기 위해 적용될 수 있습니다 . 예를 들어, 브라우저 환경에서 바인딩 prompt은 사용자 입력을 요청하는 작은 대화 상자를 표시하는 기능을 보유합니다. 다음과 같이 사용됩니다.

prompt("Enter passcode");

함수를 실행하는 것을 invoking, calling 또는 applying라고 합니다. 함수 값을 생성하는 표현식 뒤에 괄호를 넣어 함수를 호출할 수 있습니다. 일반적으로 함수를 보유하는 바인딩의 이름을 직접 사용합니다. 괄호 안의 값은 함수 내부의 프로그램에 제공됩니다. 예에서 prompt 함수는 대화 상자에 표시할 텍스트로 제공한 문자열을 사용합니다. 함수에 주어진 값을 arguments라고 합니다. 함수에는 다른 수 또는 다른 유형의 인수가 필요할 수 있습니다.

prompt 함수는 대부분의 결과 대화 상자가 표시되는 방식을 제어할 수 없기 때문에 최신 웹 프로그래밍에서는 많이 사용되지 않지만 연습에 도움이 될 수 있습니다.

console.log 함수

예에서는 값을 출력하는데 console.log를 사용했습니다. 대부분의 JavaScript 시스템은 (모든 최신 웹 브라우저 및 Node.js 포함) 인수를 일부 텍스트 출력 장치에 기록하는 console.log 함수를 제공합니다. 브라우저에서 출력은 JavaScript 콘솔에 표시됩니다. 브라우저 인터페이스의 이 부분은 기본적으로 숨겨져 있지만 대부분의 브라우저는 F12를 누르거나 Mac의 경우 command-option-I를 누르면 열립니다. 그래도 작동하지 않으면 메뉴를 통해 Developer Tools 또는 이와 유사한 항목을 검색하십시오.

본 강좌에서 예(또는 자신의 코드)를 실행할 때 브라우저의 JavaScript 콘솔 대신 예 다음에 console.log 출력이 표시됩니다.

let x = 30;
console.log("the value of x is", x);
// → the value of x is 30

바인딩 이름에는 마침표 문자가 포함될 수 없지만 console.log에는 마침표가 있습니다. 그 이유는 단순 바인딩이 아니기 때문입니다. 실제로 바인딩이 보유한 값에서 속성 log를 검색하는 표현식입니다. 이것이 정확히 무엇을 의미하는 지에 관해서는 4장에서 소개합니다.

Return 값

대화 상자를 표시하거나 화면에 텍스트를 쓰는 것은 바람직하지 않습니다. 함수는 값을 생성할 수 있는데 이 경우 유용하게 사용하기 위해 return을 사용합니다. 예를 들어, 함수 Math.max는 숫자 인수의 양에 관계없이 가장 큰 인수를 반환합니다.

console.log(Math.max(2, 4));
// → 4

함수가 값을 생성하면 해당 값을 반환한다고 합니다. 값을 생성하는 모든 것은 JavaScript의 표현식입니다. 즉, 더 큰 표현식 내에서 함수 호출을 사용할 수 있습니다. 여기서 Math.max의 반대인 Math.min에 대한 호출은 더하기 표현식의 일부로 사용됩니다.

console.log(Math.min(2, 4) + 100);
// → 102

제어 흐름

프로그램에 하나 이상의 명령문이 포함되어 있으면 명령문은 마치 하나의 이야기인 것처럼 위에서 아래로 실행됩니다. 이 예제 프로그램에는 두 개의 명령문이 있습니다. 첫 번째는 사용자에게 숫자를 묻고 첫 번째 다음에 실행되는 두 번째는 해당 숫자의 제곱을 보여줍니다.

let theNumber = Number(prompt("Pick a number"));
console.log("Your number is the square root of " +
            theNumber * theNumber);

함수 Number는 값을 숫자로 변환합니다. prompt의 결과가 문자열 값이고 숫자가 필요하기 때문에 해당 변환이 필요합니다. 유사한 함수로 String과 Boolean은 값을 해당 타입으로 변환합니다.

다음은 직선 제어 흐름의 다소 간단한 도식 표현입니다.

조건부 실행

모든 프로그램이 직선 도로는 아닙니다. 예를 들어, 프로그램이 당면한 상황에 따라 적절한 분기를 취하는 분기 도로를 만들고 싶을 수 있습니다. 이것을 조건부 실행이라고 합니다.

조건부 실행은 if 키워드로 생성됩니다. 간단한 경우에 특정 조건이 유지되는 경우에만 일부 코드가 실행되기를 원합니다. 예를 들어 입력이 실제로 숫자인 경우에만 입력의 제곱을 표시할 수 있습니다.

let theNumber = Number(prompt("Pick a number"));
if (!Number.isNaN(theNumber)) {
  console.log("Your number is the square root of " +
              theNumber * theNumber);
}

이러한 수정과 관련하여 "parrot"을 입력하면 출력이 표시되지 않습니다.

if 키워드는 부울 식의 값에 따라 문장을 실행하거나 건너 뜁니다. 결정식은 키워드 뒤, 괄호 사이에 작성하고 그 뒤에 실행할 문이 옵니다.

Number.isNaN 함수는 제공된 인수가 NaN인 경우에만 true를 반환하는 표준 JavaScript 함수입니다. Number 함수는 타당한 수를 나타내지 않는 문자열이 주어지는 경우에 NaN을 반환하기 때문에 조건은 "unless theNumber is not-a-number, do this"로 번역됩니다.

if 뒤에 오는 문장은 중괄호로 묶입니다. 중괄호를 사용하여 블록 이라고 하는 단일 명령문으로 여러 명령문을 그룹화할 수 있습니다. 이 경우에는 하나의 명령문만 보유하므로 생략할 수도 있지만 필요한지 여부를 생각하지 않아도 되도록 대부분의 JavaScript 프로그래머는 이와 같이 모든 래핑된 명령문에서 사용합니다.

if (1 + 1 == 2) console.log("It's true");
// → It's true

조건이 참일 때 실행되는 코드뿐만 아니라 다른 경우를 처리하는 코드도 있는 경우가 많습니다. 다음 대체 경로는 다이어그램에서 두 번째 화살표로 표시됩니다. else 키워드를 if와 함께 사용하여 두 개의 개별 대체 실행 경로를 생성할 수 있습니다.

let theNumber = Number(prompt("Pick a number"));
if (!Number.isNaN(theNumber)) {
  console.log("Your number is the square root of " +
              theNumber * theNumber);
} else {
  console.log("Hey. Why didn't you give me a number?");
}

선택할 수 있는 경로가 두 개 이상인 경우 여러 if/else 쌍을 함께 연결할 수 있습니다.

let num = Number(prompt("Pick a number"));

if (num < 10) {
  console.log("Small");
} else if (num < 100) {
  console.log("Medium");
} else {
  console.log("Large");
}

위 프로그램의 스키마는 다음과 같습니다.

while 및 do 루프

0에서 12까지의 모든 짝수를 출력하는 프로그램은 다음과 같이 작성할 수 있습니다.

console.log(0);
console.log(2);
console.log(4);
console.log(6);
console.log(8);
console.log(10);
console.log(12);

예를 들어, 1000보다 작은 모든 짝수가 필요한 경우 이러한 접근 방식은 바람직하지 않습니다. 필요한 것은 코드 조각을 여러 번 실행하는 방법입니다. 이러한 형태의 제어 흐름을 loop라고 합니다.

제어 흐름을 반복하면 이전에 있었던 프로그램의 특정 지점으로 돌아가 현재 프로그램 상태로 이를 반복할 수 있습니다. 이것을 계산하는 바인딩과 결합하면 다음과 같이 할 수 있습니다.

let number = 0;
while (number <= 12) {
  console.log(number);
  number = number + 2;
}
// → 0
// → 2
//   … etcetera

키워드 while로 시작하는 명령문은 루프를 생성합니다. 단어 while 다음에는 괄호 안의 표현식이 오고 표현식이 부울로 변환될 때 제공되는 값을 생성하는 한 루프는 해당 명령문을 계속 입력합니다.

number 바인딩은 프로그램의 진행 상황을 추적할 수 있는 방식을 보여줍니다. 루프가 반복될 때마다 number는 이전 값보다 둘 더 많은 값을 가져 옵니다. 모든 반복이 시작될 때 숫자 12와 비교하여 프로그램의 작업이 완료되었는지 여부를 결정합니다.

실제 유용한 작업으로서 210의 값을 계산하고 보여주는 프로그램을 작성할 수 있습니다. 두 개의 바인딩을 사용합니다. 하나는 결과를 추적하기 위한 것이고 다른 하나는 이 결과에 2를 곱한 횟수를 계산하는 것입니다. 루프는 두 번째 바인딩이 아직 10에 도달했는지 여부를 테스트하고 그렇지 않은 경우 두 바인딩을 모두 업데이트합니다.

let result = 1;
let counter = 0;
while (counter < 10) {
  result = result * 2;
  counter = counter + 1;
}
console.log(result);
// → 1024

do 루프는 while 루프와 유사한 구조입니다. do 루프는 항상 본문을 최소한 한 번 이상 실행하고 첫 번째 실행 후에만 중지해야 하는지 여부를 테스트하기 시작합니다. 이를 반영하기 위해 테스트는 루프 본문 뒤에 나타납니다.

let yourName;
do {
  yourName = prompt("Who are you?");
} while (!yourName);
console.log(yourName);

이 프로그램은 이름을 입력하도록 합니다. 빈 문자열이 아닌 것을 얻을 때까지 계속해서 묻습니다. ! 연산자를 적용하면 값을 부정하기 전에 부울 타입으로 변환됩니다. ""을 제외한 모든 문자열이 true로 전환됩니다. 이것은 비어 있지 않은 이름을 제공할 때까지 루프가 계속 순환한다는 것을 의미합니다.

코드 들여쓰기

블록 내에서 이 들여쓰기의 역할은 코드의 구조를 눈에 띄게 만드는 것입니다. 다른 블록 내에서 새 블록이 열리는 코드에서는 한 블록이 끝나고 다른 블록이 시작되는 위치를 파악하기 어려울 수 있습니다. 적절한 들여쓰기를 사용하면 프로그램의 시각적 모양이 내부 블록의 모양과 일치합니다. 저는 열린 블록마다 두 개의 공백을 사용하는 것을 좋아하지만 취향이 다릅니다. 어떤 사람은 4칸을 사용하고 어떤 사람은 탭 문자를 사용합니다. 중요한 것은 각각의 새 블록이 동일한 양의 공간을 추가한다는 것입니다.

if (false != true) {
  console.log("That makes sense.");
  if (1 < 2) {
    console.log("No surprise there.");
  }
}

for 루프

이 패턴은 매우 일반적이기 때문에 JavaScript 및 이와 유사한 언어는 약간 더 짧고 포괄적인 형식인 for 루프를 제공합니다.

for (let number = 0; number <= 12; number = number + 2) {
  console.log(number);
}
// → 0
// → 2
//   … etcetera

for 키워드 뒤의 괄호에는 두 개의 세미콜론이 포함되어야 합니다. 첫 번째 세미콜론 앞의 부분은 일반적으로 바인딩을 정의하여 루프를 초기화 합니다. 두 번째 부분은 루프가 계속되어야 하는지 여부를 확인하는 표현식입니다. 마지막 부분은 모든 반복 후에 루프의 상태를 업데이트 합니다. 대부분의 경우 이것은 while 구문보다 짧고 명확합니다.

let result = 1;
for (let counter = 0; counter < 10; counter = counter + 1) {
  result = result * 2;
}
console.log(result);
// → 1024

루프 벗어나기

break라는 특수 구문은 루프에서 즉시 점프하는 효과가 있습니다.

다음 프로그램은 break 명령문을 보여줍니다. 20보다 크거나 같고 7로 나누어 떨어지는 첫 번째 숫자를 찾습니다.

for (let current = 20; ; current = current + 1) {
  if (current % 7 == 0) {
    console.log(current);
    break;
  }
}
// → 21

키워드 continue는 break와 유사합니다. continue가 루프 본문에서 발생하면 제어가 본문 밖으로 이동하고 루프의 다음 반복을 계속합니다.

간결한 바이딩 갱신

특히 루핑할 때 프로그램은 바인딩의 이전 값을 기반으로 한 값을 유지하기 위해 바인딩을 갱신해야 하는 경우가 많습니다.

counter = counter + 1;

JavaScript는 이에 대한 바로 가기를 제공합니다.

counter += 1;

비슷한 단축키 result *= 2는 두 배로 계산하거나 counter -= 1을 통해 아래로 계산하는 것과 같은 다른 많은 연산자에서도 작동합니다.

for (let number = 0; number <= 12; number += 2) {
  console.log(number);
}

counter += 1과 counter -= 1은 counter++ 및 counter--과 같습니다.

switch 사용

보통 다음과 같은 코드가 작성됩니다.

if (x == "value1") action1();
else if (x == "value2") action2();
else if (x == "value3") action3();
else defaultAction();

switch를 사용하여 표현할 수도 있습니다.

switch (prompt("What is the weather like?")) {
  case "rainy":
    console.log("Remember to bring an umbrella.");
    break;
  case "sunny":
    console.log("Dress lightly.");
  case "cloudy":
    console.log("Go outside.");
    break;
  default:
    console.log("Unknown weather type!");
    break;
}

2차원으로 작업하려면 루프 내부에 루프가 필요합니다. 두 루프의 몸체 주위에 중괄호를 두어 시작과 끝을 쉽게 알 수 있다. 이 몸체를 적절하게 들여쓴다. 루프의 순서는 문자열을 구성하는 순서를 따라야 한다(라인별로, 왼쪽에서 오른쪽, 위에서 아래로). 따라서 외부 루프는 라인을 처리하고 내부 루프는 라인의 문자를 처리한다.

진행 상황을 추적하려면 두 개의 바인딩이 필요하다. 주어진 위치에 공백이나 해시 기호를 넣을지 여부를 알기 위해 두 카운터의 합이 짝수(% 2)인지 테스트할 수 있다.

라인 바꿈 문자를 추가하여 라인을 종료하는 것은 줄이 구성된 후에 발생해야 하므로 내부 루프 이후에 외부 루프 내부에서 수행한다.
